栈，队列和双端队列

发表于 2020-06-14

栈

栈是最简单的数据结构，先进后出。
抽象数据类型(ADT)：

  • S.push(e):将一个元素e添加到栈S的栈顶
  • S.pop():将栈顶元素移除并返回栈顶的元素，如果此时栈为空的，操作将出错。
  • S.top():不移除栈顶元素，返回一个栈S的栈顶元素；若为空，操作将出错。
  • S.is_empty() : 如果栈中不包含任何元素，返回布尔值“True”
  • len(s): 返回栈S中元素的数量。用 __ len __ 这个方法实现它。

队列

队列：先进先出，允许插入的一端为队尾，允许删除的一端为队头。
队列的抽象数据类型（ADT）支持：

  • Q.enqueue(e) : 向队列Q的队尾添加一个元素。
  • Q.dequeue() :向队列Q中移除并返回第一个元素，如果队列为空，则触发一个错误。
  • Q.first() :在不移除的前提下返回第一个元素，如果队列为空，则触发一个错误。
  • Q.is_empty() :如果队列Q没有包含任何元素则返回“True”
  • len(Q) : 返回Q中元素的数量；使用 __ len __ 方法实现。

基于数组的队列实现：

1：到一个元素退出队列时，我们可以使用pop(0)从列表中移除第一个元素，但其最低效，因为删除列表首元素后，列表后面的元素都会左移动一位，为了填补pop操作给序列造成的“洞”，因此pop(0)的操作的调用总是处于最坏的情况，耗时为O(n)。

2：对其改进避免调用pop(0),可以用一个指针为空的指针代替这个数组中离队的元素，并保留一个变量来储存当前在最前面的元素的索引，这样对离队操作而言耗时O(1).

3:但不幸的是，修改后的方法仍然有一个缺点。如果不断重复地往一个队列中添加一个新的元素，然后删除最右端（将列表首元素指向空指针），其底层的列表大小将会增长到O(m)，m为自队列建立以来对队列进行追加元素操作的数量总和，而不是当前队列中元素的数量。对于这种情况，使用循环数组，假设底层数组的长度最小固定为N，当元素超过其数值时，底层数组长度变为2N，当元素不足N时，将其数组循环 以增加和删除操作，并保存首元素的索引。 当其删除操作致使实际元素小于底层数组大小的1/4 时，将底层数组减半（但最小数组固定为N）

实现方法：

  • _data: 指向一个固定容量的列表实例。
  • _size: 是一个整数，代表当前存储在队列内的元素的数量。(与_data 列表的长度正好相对）
  • _front: 代表_data实例队列中第一个元素的索引。当因增加或删除导致底层数组进行扩展或收缩时，将队列的—_front索引初始化为0.

双端队列

其数据结构：支持在队列的头部和尾部都进行插入和删除操作。

  • D.add_first(e) :向双端队列前面添加一个元素e
  • D.add_last(e) :向双端队列的后面添加一个元素e
  • D.delete_first():从双端队列中移除并返回第一个元素。若双端队列为空，则触发一个错误。
  • D.delete_last() :从双端队列中移除并返回最后一个元素，若双端队列为空，则触发一个错误。
  • D.first() : 返回（但不移除）双端队列的第一个元素。若双端队列为空，则触发一个错误。
  • D.last() : 返回（但不移除）双端队列的最后一个元素，若双端队列为空，则触发一个错误。
  • D.is_empty() :如果双端队列不包含任何一个元素，则返回布尔值True
  • len(D) : 返回当前双端队列的元素个数，用 __ len __ 方法实现。

使用循环数组实现双端队列：建议保持三个变量： _data , _size , _front
